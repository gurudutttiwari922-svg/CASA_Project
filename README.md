# CASA — Constraint-Based Academic Scheduling and Allocation System

## Overview
CASA is a smart academic scheduling system designed to manage timetables, handle dynamic teacher availability, and automate class reallocation using constraint-based logic.

## Features (Phase 2)
- Constraint-based scheduling engine
- Teacher availability tracking
- Automatic class reallocation
- Conflict detection (teacher, room, time)
- Reallocation logging system

## Vision (Phase 3)
- Persistent timetable storage (database)
- Multi-user login system (Admin, HOD, CC, Teacher)
- Attendance-based automatic reallocation
- Approval workflow system
- Teacher directory with live status
- Room/venue booking system with conflict management

## Project Structure
```
src/
└── com/casa/
    ├── core/        # Scheduling logic engine
    ├── model/       # Data models (Teacher, Room, Session, etc.)
    ├── main/        # Entry point (MainApp)
```
